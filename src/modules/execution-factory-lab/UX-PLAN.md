# 能力库（Lab）用户体验优化计划

> 在 Phase D 能力补齐完成后，从「能用」升级到「好用」。不切换菜单/目录，不改变 BFF 契约。

## 迭代 1 — 主链路闭环

**业务目标**：降低「创建后不知道下一步」的流失；减少详情页误触编辑；让列表可扫读。

| # | 功能 | 用户价值 | 验收 |
|---|------|----------|------|
| 1.1 | 创建成功 Toast + 自动打开详情 | 创建即进入治理流程 | 各创建抽屉成功后打开详情 |
| 1.2 | 详情默认只读，点击「编辑」才改 | 查看 ≠ 编辑，减少误操作 | 概览 Tab 默认文本展示 |
| 1.3 | 操作分组：发布 / 编辑 / 更多(导出·危险) | 降低按钮墙认知负担 | 危险操作收进「更多」 |
| 1.4 | 卡片：类型色 + 状态左边条 + 新建高亮 | 列表扫一眼能区分状态 | 创建后 3s 高亮 |
| 1.5 | 空状态：真空 vs 筛选无结果 | 用户知道是删筛选还是去创建 | 两种文案 + 清除筛选 |
| 1.6 | 筛选记忆（kind/status） | 回访不用重选 | localStorage 持久化 |

## 迭代 2 — 降门槛

**业务目标**：非开发用户也能完成接入；减少 JSON/手工填写的挫败感。

| # | 功能 | 用户价值 | 验收 |
|---|------|----------|------|
| 2.1 | 「添加能力」菜单分组 + 一行说明 | 知道选哪条路径 | API / 扩展 / 高级 三组 |
| 2.2 | MCP/Skill 创建表增加分类 Select | 与市场/治理分类一致 | 调用 GET /categories |
| 2.3 | 调试 Tab：MCP 工具下拉 | 不用猜 tool 名 | 来自 tools 列表 |
| 2.4 | 调试 Tab：高级 JSON 折叠 | 简单场景不暴露 JSON | Collapse 默认关 |
| 2.5 | Function 调试示例 payload 一键填充 | 降低空白页焦虑 | 按钮填充 `{"x":1}` |
| 2.6 | 保存/发布成功 Toast | 操作有反馈 | message.success |

## 迭代 3 — 信任与联动

**业务目标**：权限可理解；实验版可放心试用；市场与能力库形成闭环。

| # | 功能 | 用户价值 | 验收 |
|---|------|----------|------|
| 3.1 | 无权限时 disabled + Tooltip | 不是「功能缺失」 | LabPermissionHint |
| 3.2 | 实验 Badge Tooltip 说明 | 消除「不敢用」 | hover 解释双轨 |
| 3.3 | 空能力库 CTA「去市场看看」 | 冷启动有引导 | 链到 catalog |
| 3.4 | 市场安装后跳转能力库并打开详情 | 安装即验收 | navigate + open detail |
| 3.5 | 发布/下线二次确认 | 防误触状态变更 | Popconfirm |
