# 知识网络迁移计划

## 1. 执行原则

- 先对照旧 `vega` 页面矩阵，再写代码。
- 每一刀都要同时检查：页面结构、关键操作、弹窗/抽屉、空态/错误态、视觉层级。
- 不再按“用户指出一点就修一点”的方式推进。
- 未完成的差异必须写回差异报告。

## 2. 阶段划分

## Phase 0：迁移基线

目标：

- 建立模块骨架
- 接管一级菜单“领域知识网络”
- 建立 `spec / plan / tasks`
- 建立页面迁移矩阵、视觉基线、差异报告

状态：

- 已完成

## Phase 1：工作区主链路

目标：

- 知识网络列表
- 创建/编辑
- 导入入口占位
- 主工作区壳
- 概览
- 预览基础壳
- 概念分组
- 对象类第一阶段
- 关系类第一阶段
- 行动类第一阶段

状态：

- 进行中

当前剩余：

- 对象类详情 / 索引设置
- 关系类详情 / 完整映射规则
- 行动类详情 / 执行映射基础
- 导入链路从占位进入可执行状态

## Phase 2：扩展资源能力

目标：

- 指标
- 任务
- 预览增强
- 工作区更完整的查询与操作区

状态：

- 未开始

## Phase 3：深能力补齐

目标：

- 对象类索引设置完整实现
- 关系类高级映射配置
- 行动类执行计划 / 日志 / 取消执行

状态：

- 未开始

## 3. 当前强制执行顺序

在进入 Phase 2 前，必须先完成下面 4 件事：

1. 对象类详情页
2. 关系类详情页
3. 行动类详情页
4. 第一轮整体 UI 对齐

原因：

- 当前资源页已经有列表和编辑入口，但详情链路还不闭环。
- 如果现在直接做指标和任务，会继续扩大范围，进一步稀释样式和交互对齐质量。

## 4. 视觉与交互收口计划

在详情页之后，必须做一轮统一收口，至少覆盖：

- 工作区左侧导航选中态
- 标题区、内容区、卡片区留白
- 列表工具栏
- 列表行内操作
- 标签、状态标签、空态
- 抽屉/详情页信息分组

收口时以 [visual-baseline.md](D:/openbkn/bkn-studio/specs/knowledge-network-migration/visual-baseline.md) 为准。

## 5. 编码前后检查

每个切片开始前：

- 核对迁移矩阵里对应页面
- 核对差异报告里对应缺口

每个切片结束后：

- 更新差异报告
- 执行 `lint / test / build`
- 标记该页面是“结构完成”还是“结构 + 视觉完成”
