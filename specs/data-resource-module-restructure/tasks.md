# 数据连接与数据目录重构任务清单

## T1. 信息架构调整

- [ ] 明确一级菜单为“数据连接”“数据目录”
- [ ] 明确二级菜单和页面入口
- [ ] 统一页面标题、面包屑、导航文案
- [ ] 梳理旧入口到新入口的迁移关系

## T2. 路由与页面归位

- [ ] 设计连接侧路由结构
- [ ] 设计目录侧路由结构
- [ ] 将扫描任务归位到连接侧
- [ ] 将资源详情入口归位到目录侧

## T3. 数据连接页面收敛

- [ ] 收敛 catalog 列表页为连接列表
- [ ] 收敛 catalog 详情页为连接详情
- [ ] 保留连接测试、启停、扫描相关动作
- [ ] 移除不属于连接层的资源治理动作

## T4. 数据目录页面建设

- [ ] 建立资源列表页
- [ ] 建立资源详情页
- [ ] 提供按连接、类型、状态筛选
- [ ] 提供逻辑视图筛选或二级入口

## T5. 资源治理能力迁移

- [ ] 数据查询迁入资源详情
- [ ] 数据向量化迁入资源详情
- [ ] 数据质量迁入资源详情
- [ ] 数据语义理解迁入资源详情

## T6. 后端接口适配确认

- [ ] 确认 `/catalogs` 接口族满足连接页需求
- [ ] 确认 `/resources` 接口族满足目录页需求
- [ ] 确认 discover task 接口仅归属连接侧
- [ ] 确认 build task 接口仅归属资源侧
- [ ] 评估是否需要补充资源聚合查询接口

## T7. 验收

- [ ] 用户可区分连接管理和资源管理
- [ ] 扫描相关操作只出现在连接侧
- [ ] 查询/索引/质量/语义只出现在资源侧
- [ ] 逻辑视图不再作为独立根模型与资源体系并列
- [ ] 页面结构与后端 `Catalog / Resource` 模型一致
