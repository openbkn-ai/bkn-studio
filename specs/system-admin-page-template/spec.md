# 系统管理页面模板规范

## 1. 背景

系统管理模块已经形成了相对稳定的企业级业务风格，核心特征是：

- 直角风格，不使用大圆角卡片式页面
- 顶部操作栏 + 页面说明 + 表格/表单主体的清晰分层
- 列表页、新建页、抽屉页采用同一套视觉语言
- 页面职责单一，不在一个页面里混杂多套视图模式

前期页面问题主要集中在：

- 页面结构不统一，某些页面自成一套
- 交互状态过多，导致页面复杂度失控
- 分页、筛选、抽屉、新建流程缺少统一模板
- 公共状态钩子使用不规范，容易引发持续重渲染

因此需要把当前系统管理已经验证有效的页面风格，提炼成后续可复用的模板规范。

## 2. 适用范围

本规范适用于 `studio/system/*` 下的页面，优先覆盖：

- 用户管理
- 角色管理
- 授权管理
- 日志管理

并作为后续其他后台业务模块的参考模板。

## 3. 页面模板分类

### 3.1 标准列表页

适用场景：

- 管理对象以列表为主
- 支持搜索、筛选、分页、批量/单条操作

标准结构：

1. 页面容器
   - 白色背景
   - 直角边界
   - 不使用浮层式大圆角卡片

2. 顶部操作栏
   - 左侧：主操作按钮、刷新按钮
   - 中间：页面说明或业务提示
   - 右侧：搜索、筛选项

3. 说明提示区
   - 用浅色提示条承载业务说明
   - 不与表格直接混在一起

4. 列表主体区
   - 表头统一浅灰底
   - 表格内容不加粗
   - 操作列使用文本链接，不使用大量按钮堆叠

5. 分页区
   - 统一放在表格下方
   - 与表格保留间距
   - 使用统一分页组件风格

### 3.2 标准新建页

适用场景：

- 需要分步骤选择对象、配置参数、提交保存

标准结构：

1. 顶部操作栏
   - 返回按钮
   - 页面标题
   - 页面副说明
   - 主提交按钮

2. 摘要信息区
   - 展示当前选择结果或进度
   - 适合用 2-4 个摘要块展示

3. 主体分区
   - 每个业务步骤独立成一个直角分区
   - 分区包含标题、说明、输入区域
   - 禁止把多个步骤揉成一个无边界的大表单

4. 空状态
   - 当上游未选择时，用空状态或提示文案说明依赖关系
   - 不直接展示“不可用但仍可操作”的控件堆

### 3.3 标准抽屉页

适用场景：

- 详情查看
- 局部编辑
- 对象级配置

标准结构：

1. 抽屉头部
   - 标题明确
   - 直角风格
   - 只保留必要操作

2. 对象摘要区
   - 展示对象名称、类型、关键状态
   - 可补充少量统计信息

3. 分区内容区
   - 新增区
   - 已有配置区
   - 历史/说明区

4. 空状态与只读状态
   - 无数据时明确展示空状态
   - 不可编辑对象要用锁定说明，而不是让控件看起来像坏掉

## 4. 视觉与交互规则

### 4.1 视觉规则

- 页面、分区、抽屉、弹窗全部使用直角
- 页面主背景为白色，外层工作区为浅灰
- 表头使用浅灰底色
- 正文默认字号 `14px`
- 辅助信息使用 `13px`
- 标签使用 `12px`
- 列表内容默认常规字重，不加粗
- 标题与关键数字可以加粗，但范围要克制

### 4.2 表格规则

- 表头可以加粗
- 表格内容默认不加粗
- 操作列优先使用文本链接
- 状态优先使用纯文本或轻量标签
- 列表项名称与副信息上下分层显示

### 4.3 筛选区规则

- 搜索、筛选统一放在操作栏右侧
- 筛选项不做大圆角包裹容器
- 筛选项之间的间距统一

### 4.4 分页规则

- 分页使用统一公共组件
- 样式统一参考系统管理当前已收敛版本
- 默认支持：
  - 总数
  - 页码切换
  - 每页条数切换
- 禁止每个页面单独重写一套分页样式

### 4.5 状态管理规则

- 分页状态、关键词状态使用公共钩子
- 公共钩子返回的更新函数必须稳定
- 页面里禁止写出会导致持续重渲染的 `useEffect + setState` 回路
- 页面不允许混合多套互斥视图模式，除非业务明确需要

## 6. 禁止事项

后续页面建设中，禁止出现以下情况：

1. 把一个后台页面做成圆角营销风格
2. 在一个页面里并行堆叠多套主视图模式
3. 表格、分页、筛选、抽屉各写各的样式
4. 通过页面局部补丁不断叠加复杂状态
5. 在 `useEffect` 中反复触发分页或筛选状态更新，形成渲染回路
6. 把提示信息、表格、表单全部揉成一个无边界的大面板

## 7. 后续使用方式

后续新建系统管理页时，必须按以下顺序执行：

1. 先确定页面属于列表页、新建页还是抽屉页
2. 直接复用模板结构，不重新发明布局
3. 复用公共分页和公共状态钩子
4. 若页面需要扩展，只允许在模板基础上增量调整
5. 如确实需要突破模板，必须先补充规范说明
