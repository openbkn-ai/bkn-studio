# Issue 39 当前阶段实施方案

## 目标

把对象详情页增强收敛为两个小目标：

1. 让用户看见对象类映射到哪个数据视图
2. 让用户在对象详情页内直接查看和查询该数据视图的数据

## 实施范围

### 本次纳入

- 对象类与数据视图关系展示
- 数据视图样本数据预览
- 搜索 / 过滤
- 分页

### 本次不纳入

- 多数据视图映射
- SQL 编辑器
- 保存为 Skill
- 关系探索
- 保存为 Logic

## 方案拆解

### Phase 1: 对象类与数据视图关系展示

#### 前端范围

- 改造 `ObjectTypeDetailScene`
- 新增对象详情“绑定数据视图”区块

建议新增组件：

- `ObjectTypeBoundViewPanel`

#### 数据来源

优先复用：

- `ObjectTypeDetail.dataSource`

#### 区块展示建议

- 数据视图名称
- 资源 ID
- catalog / data source 标识
- 跳转到数据目录资源详情入口

#### 空状态

- 无绑定数据视图时，显示清晰空状态提示

### Phase 2: 数据查询能力

#### 前端范围

新增组件：

- `ObjectTypeDataQueryPanel`

#### 服务层范围

优先复用：

- `getObjectTypeResourcePreview`

#### 交互能力

- 默认展示样本数据
- 支持关键字搜索
- 支持分页

#### 查询策略

第一版建议：

- 先采用样本数据预览 + 前端过滤
- 不引入 SQL 语法输入

这样实现风险最低，也符合当前阶段收敛目标

## 页面结构建议

对象详情页建议调整为：

1. 对象摘要
2. 数据属性 / 逻辑属性
3. 绑定数据视图
4. 数据查询

说明：

- 属性区块继续保留，不做大改
- 数据视图和数据查询作为新增能力独立落位

## 组件边界建议

- `ObjectTypeDetailScene`
  - 顶层加载和布局
- `ObjectTypePropertyPanel`
  - 数据属性 / 逻辑属性
- `ObjectTypeBoundViewPanel`
  - 绑定数据视图信息
- `ObjectTypeDataQueryPanel`
  - 样本数据预览、搜索、分页

## 风险与依赖

### 前端风险

- 对象详情页复杂度会上升

应对：

- 按子面板拆分，不把所有逻辑继续堆在一个场景文件里

### 服务依赖

需要确认：

1. `dataSource` 中是否有足够字段支持数据目录跳转
2. 资源预览接口是否稳定
3. 预览返回的数据量是否足够支持前端过滤体验

## 后续需求预留

以下内容明确留到后续需求处理：

- SQL 查询工作台
- 查询沉淀为 Skill
- 关系探索
- 探索结果沉淀为 Logic / Skill

## 验证策略

### UI 验证

- 对象已绑定数据视图
- 对象未绑定数据视图
- 数据预览为空
- 长表格分页显示稳定

### 服务验证

- 预览接口失败时有错误提示
- catalog 权限不足时有明确反馈

### 集成验证

- 可从对象详情跳转到数据目录资源详情
- 对象详情数据查询区块与属性区块不互相干扰
