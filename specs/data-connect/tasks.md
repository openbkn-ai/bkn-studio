# 数据连接任务拆分

## 前端任务

### FE-1 新建数据连接模块骨架

- 新建 `src/modules/data-connect/*`
- 新增路由入口
- 左侧菜单挂载“数据连接”

### FE-2 实现数据源列表页

- 列表表格
- 搜索
- 类型筛选
- 排序
- 新建按钮
- 行操作菜单
- 空态、错态、加载态

### FE-3 实现数据源详情抽屉

- 基础信息展示
- 连接配置展示
- 更新时间、更新人

### FE-4 实现数据源新建页

- 连接器类型选择
- 第二步配置表单
- 动态字段显隐
- 校验规则
- 保存提交

### FE-5 实现数据源编辑页

- 回填数据
- 密码脱敏逻辑
- 更新提交

### FE-6 实现测试连接

- 表单内触发测试连接
- 结果反馈

### FE-7 补齐 service / types / i18n

- `data-connect.service.ts`
- `data-connect.ts`
- 中英文文案

## 后端任务

### BE-1 新增数据连接模块接口定义

- 定义 service interface
- 定义 access interface
- 定义请求响应模型

### BE-2 新增数据连接 handler 与路由

- 在 `driveradapters/routers.go` 注册新路由
- 实现 handler
- 补 handler test

### BE-3 实现连接器列表接口

- 返回连接器列表
- 明确字段：显示名、原始类型、协议、分类

### BE-4 实现数据源 CRUD

- 列表查询
- 详情查询
- 创建
- 更新
- 删除

### BE-5 实现测试连接接口

- 接收动态连接参数
- 调用底层访问层完成连接测试
- 返回成功/失败结果

### BE-6 接入底层访问层

- 决定走新 access 还是复用 `vega_backend_access`
- 封装与底层服务的交互

## 联调任务

### INT-1 对齐接口契约

- 连接器字段
- 数据源详情结构

### INT-2 对齐动态字段逻辑

### INT-3 对齐错误处理

### QA-1 页面验证

- 列表页可进入
- 搜索、筛选、排序正常
- 空态与错误态正常

### QA-2 表单验证

- 新建流程正常
- 编辑回填正常
- 动态字段切换正常
- 校验规则生效

### QA-3 接口验证

- CRUD 接口返回符合契约
- 测试连接可正确反馈

### QA-4 工程验证

- `pnpm lint`
- `pnpm test -- --run`
- `pnpm build`
- 后端单测或至少关键 handler/service 自测

## 当前推荐起始切片

建议先做“第一阶段最小闭环”：

1. 连接器列表接口
2. 数据源列表接口
3. `bkn-studio` 数据连接列表页

原因：

- 这是最小可见成果
- 能先打通菜单、路由、列表和后端返回结构
- 也能尽快验证新模块在当前壳层中的接入方式
