# 应用壳层开发计划

## 目标

把当前项目从“单列页面容器”升级成“控制台式应用壳层”，先完成顶部栏、左侧导航和主工作区三大结构。

## 实施范围

### 计划改动的前端区域

- `src/app/router/shell.tsx`
- `src/app/router/create-router.tsx`
- `src/styles/global.css`
- `src/framework/ui/*`
- 如有必要，新增壳层相关静态配置文件

### 计划改动的后端区域

- 无

## 当前实现基线

当前 `AppShell` 已具备：

- 全局 header
- 路由内容承载位
- Ant Design provider 下沉

当前缺失：

- 正式品牌头部结构
- 左侧导航
- 工作区顶部结构
- 统一布局样式规范

## 开发切片

### 切片 1：顶部栏骨架

目标：

- 将现有简单 header 升级为正式全局顶部栏
- 定义品牌区、预留导航区、右侧操作区

影响范围：

- `src/app/router/shell.tsx`
- 顶部栏相关样式

### 切片 2：左侧导航骨架

目标：

- 新增固定宽度左侧导航
- 支持菜单分组、展开、选中态
- 使用本地静态菜单配置先跑通结构

影响范围：

- `src/app/router/shell.tsx`
- 导航组件和菜单配置
- 导航相关样式

### 切片 3：主工作区骨架

目标：

- 在右侧主区域内补齐面包屑区、标题区、工具栏区、内容区
- 约定后续页面如何注入这些区块

影响范围：

- `src/app/router/shell.tsx`
- `src/framework/ui/*`
- 全局样式

### 切片 4：模块接入验证

目标：

- 用现有 `starter` 模块验证新壳层可以正常承接页面
- 确认页面在新工作区中布局合理

影响范围：

- `src/modules/starter/*`

## 复用与新增

### 直接复用

- 现有 `AppShell`
- 路由懒加载机制
- 404 / 异常页
- 现有页面内容组件

### 计划新增或重构

- 顶部栏组件
- 左侧导航组件
- 工作区布局组件
- 菜单静态配置
- 面包屑/标题/工具栏承载位

## 风险与处理

### 风险 1：壳层和页面骨架职责混淆

风险说明：

- 容易在这一步把列表页、表单页、详情页细节一起做进去。

处理策略：

- 本阶段只做应用壳和工作区，不处理业务页面内部结构。

### 风险 2：导航结构过早绑定业务

风险说明：

- 如果直接把现有某个业务菜单写死，后续会把壳层做成业务专属实现。

处理策略：

- 菜单先使用演示型静态配置，只验证结构和交互。

### 风险 3：样式改动过大影响现有页面

风险说明：

- 新壳层可能让现有 `starter` 页面间距、宽度和滚动表现发生变化。

处理策略：

- 先让 `starter` 页面在新布局下稳定显示，再继续沉淀页面骨架。

## 完成定义

满足以下条件即可视为本需求完成：

1. 顶部栏完成。
2. 左侧导航完成。
3. 主工作区四段结构完成。
4. `starter` 页面能正常挂在新壳层中。
5. `lint / test / build` 通过。
