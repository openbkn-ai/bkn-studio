# 应用壳层任务拆分

## 前端任务

### T1 重构顶部栏

- 定义品牌区结构
- 定义右侧操作预留区
- 固定顶部高度、边界和背景

### T2 新增左侧导航

- 新增导航容器
- 支持一级分组和二级菜单
- 支持当前选中态
- 菜单数据先用本地静态配置

### T3 新增主工作区布局

- 新增工作区容器
- 划分面包屑区、标题区、工具栏区、内容区
- 统一右侧内容区的内边距和滚动规则

### T4 设计页面接入方式

- 约定页面如何向工作区注入标题、面包屑和工具栏
- 保持模块页面不直接操作壳层结构

### T5 用 starter 模块验证接入

- 保证现有 `starter` 页面可在新壳层中正常显示
- 如有必要，补最小适配

## 后端任务

- 无

## 联调任务

- 检查导航切换后主工作区内容是否随路由更新
- 检查页面在不同内容高度下的滚动行为
- 检查空页面状态下壳层是否完整可见

## 验证任务

### V1 布局验证

- 顶部栏是否固定且边界清晰
- 左侧导航是否独立滚动
- 主工作区是否清楚分区

### V2 接入验证

- 页面是否无需自建顶部和左侧结构
- 模块内容是否能稳定落在内容区

### V3 工程验证

- `pnpm lint`
- `pnpm test -- --run`
- `pnpm build`

## 建议执行顺序

1. T1
2. T2
3. T3
4. T4
5. T5
6. V1
7. V2
8. V3

## 当前推荐起始切片

建议先做 `T1 + T2 + T3`。

原因：

- 这三个切片合起来才算真正把壳层定住
- 不需要先碰业务字段
- 可以尽快用真实页面验证整体布局方向
