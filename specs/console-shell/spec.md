# 应用壳层需求说明

## 需求标识

- Feature: `console-shell`
- Status: `draft`
- Goal: 先把 `bkn-studio` 的应用壳层固定下来，统一顶部栏、左侧导航和主工作区布局，后续业务模块只接入内容区。

## 需求背景

当前项目已经具备基础路由壳层，但现状仍然是“单列内容容器”：

- 顶部只有简单标题和模式标识
- 没有左侧导航结构
- 主工作区没有明确的面包屑区、标题区和工具栏区边界

如果直接开始做业务模块，后续每个模块都会各自补布局，最终无法形成统一控制台界面。因此第一阶段必须先把壳层定下来。

## 参考界面拆解

以当前目标界面为准，壳层需要拆成以下三层：

1. 顶部栏
2. 左侧导航
3. 主工作区

这是应用级布局，不属于某个具体业务页面。

## 范围

### 包含范围

- 顶部栏整体结构
- 左侧导航结构
- 主工作区整体布局
- 面包屑承载位
- 页面标题承载位
- 工具栏承载位
- 内容区容器
- 全局留白、边界、滚动规则

### 不包含范围

- 具体业务模块内容
- 列表页字段与表格列
- 表单页字段与校验
- 详情抽屉
- 真实权限与菜单接口
- 复杂视图切换逻辑

## 布局目标

完成后页面应呈现为：

```text
AppShell
  TopBar
  Body
    SideNav
    Workspace
      BreadcrumbArea
      PageHeaderArea
      ToolbarArea
      ContentArea
```

后续任意模块都只向 `ContentArea` 注入业务内容，并按需使用标题区和工具栏区。

## 顶部栏定义

顶部栏是全局固定区域，至少包含：

- 品牌标识区
- 全局入口或导航预留区
- 用户区或右侧操作预留区

顶部栏约束：

- 高度固定
- 和下方主体区域有明确边界
- 不承载模块级业务操作
- 不随着内容区滚动

顶部栏本阶段先做结构和样式占位，不要求接真实用户信息。

## 左侧导航定义

左侧导航是应用级导航容器，至少包含：

- 一级导航分组
- 二级菜单区
- 当前选中态
- 展开/收起状态表现

左侧导航约束：

- 固定在主体左侧
- 宽度固定
- 菜单滚动独立于内容区
- 当前模块切换由路由驱动

本阶段先支持：

- 单选中态
- 分组展示
- 二级菜单展开

本阶段暂不要求：

- 完整权限过滤
- 动态菜单接口
- 折叠态动画细节

## 主工作区定义

主工作区是业务页面承载区，至少包含：

- 面包屑区
- 页面标题区
- 工具栏区
- 内容区

主工作区约束：

- 与左侧导航分栏布局
- 内容区滚动独立
- 统一内边距
- 统一最大可读宽度或内容边界
- 内容区支持空页面也能正常呈现

### 面包屑区

用于展示页面层级路径。

要求：

- 放在工作区顶部
- 风格轻量
- 支持未来通过路由或页面配置注入

### 页面标题区

用于展示当前页面标题与辅助说明。

要求：

- 与面包屑分开
- 默认支持标题和描述
- 预留右侧页面级操作位

### 工具栏区

用于承载模块级主操作和查询筛选入口。

要求：

- 支持左侧主操作区
- 支持右侧搜索/筛选/视图操作区
- 不把具体筛选字段写死在壳层

### 内容区

用于承载真正的业务页面内容。

要求：

- 白底或浅底容器风格统一
- 支持空态
- 支持未来承载表格、卡片、画布等不同内容形态

## 当前能力与差异

### Already supported

- 已有 `AppShell` 和路由级布局挂载点
- 已有独立的主内容区和路由懒加载
- 已有 404 和异常页兜底

### Needs frontend work

- 顶部栏需要从简单标题区升级为正式全局头部
- 需要新增左侧导航布局与菜单结构
- 需要把主工作区拆成面包屑区、标题区、工具栏区、内容区
- 需要统一整体背景、留白和分栏比例
- 需要约定业务模块如何把页面内容接入工作区

### Needs backend work

- 无

### Needs explicit compromise or clarification

- 顶部栏是否保留全局一级导航入口，还是仅保留品牌区和用户区
- 左侧导航是否立即支持折叠态，还是先固定展开
- 主工作区内容区默认采用满宽自由布局，还是统一卡片容器布局

## 约束

- 壳层布局实现放在 `src/app/*` 和 `src/framework/ui/*`
- 业务模块不直接管理顶部栏和左侧导航结构
- 页面级结构通过统一工作区布局承载
- 具体菜单数据可先用本地静态配置，不阻塞壳层搭建
- 文案继续走 i18n

## 验收标准

本需求完成后，应满足：

1. 顶部栏、左侧导航、主工作区形成稳定布局。
2. 任意模块页面都自动落在统一工作区内。
3. 页面层级信息可以通过面包屑区和标题区表达。
4. 工具栏区和内容区边界清晰，后续模块无需重复搭壳。
5. 当前壳层仍符合 `app / framework / modules` 分层。
